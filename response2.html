Code and Placebos

Paul Ford had an interesting article with the idea of code having an apex evolution of language. Code by itself is an interesting subject because it’s a language built on algorithms. Then with those algorithms you can construct any number of things that your knowledge permits. It’s almost like a physical material that can meld, bent, or transformed with proper research. The blue guide was pretty fun as well, even though it reminded me of the Microsoft paperclip also butting in at any given time.
The placebo affect article reminded me how easy it is to create an easy solution to a vague question. Designing is not an easy job, the effect that a product has needs to be coherent so that the consumer can use it to its full abilities, especially for a tool. Any tool should have a distinct purpose even if it is for the static interference those people felt. There were some mixed results in the items effects on those who used them. Some felt better by having something to use as a copping device and others felt the items did nothing. There is not much data on the subject so more physical evidence on how their bodies experienced electro waves would help. I’m actually interested in the way energy affects the body so this article was a good read.
